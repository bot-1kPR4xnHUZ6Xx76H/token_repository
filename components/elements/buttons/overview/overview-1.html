General guidelines

  • Use this component to trigger an action on a page or view.
  • Only use one primary button on a page or view. The remaining actions should be lower emphasis buttons. Search results pages are an exception to this rule.
  • If you need to direct the user to another page or view, then use a link. Links are for navigation, and buttons are for actions.
  • Try to avoid repeatable button labels. If labels are repeated then ensure the accessibility considerations are met.
  • Where possible, include a text label in the button. Icons are not always clearly understood by everybody.
  • Keep body copy short, expressing the action that will occur after the interaction.

 

Anatomy

Target area: The base tappable area of the button.

Leading icon: A leading icon is used to give more clarity to the button action. For example, a bell for notification sign-up. (optional)

Label: The name of the action to be performed, ideally a verb. (optional, but preferred).

Trailing icon: A trailing icon is used to denote progression. For example, an arrow. (optional)

NOTE: You should only use one icon max.

States

Default: The resting state of the button.

Hover: Indicates that interaction is available for people using pointer devices.

Focus: Indicates that interaction is available for people using non-pointer devices.

Pressed: Indicates that interaction has occurred, and the button has been pressed down.

Loading: Informs the visitor that a button has been pressed, and something is happening. Note: The button is disabled from further interaction until either the action has been completed, or the action failed to complete.

Primary Button

Primary buttons indicate the key action to be performed on a page or view. Only one primary action must be used on a page or view, except on a search results page, where multiple primary buttons may be used. An example could be a submit button associated with a form.
